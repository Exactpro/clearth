Decimal Format

You can use DecimalFormat to parse and format various numbers in any locale. Use DecimalFormat(pattern) constructor to create new formatter. The formatter consists of pattern and symbols. Symbols can be changed using specific API.

Example:
new DecimalFormat(##.##).setDecimalSeparator(,)

Structure of the pattern:

pattern positivePattern {;negativePattern}
positivePattern {prefix}integer{.fraction}{suffix}
negativePattern the same structure as for the positivePattern
prefix \\u0000..\\uFFFD - characters
suffix the same structure as for the prefix
integer '#'*'0'*'0'
fraction '0'*'#'*

Used notation:
0 is used for minimum integer digits denotion, # - for maximum.
{} means optional, * - repetitive (zero or more times)

Example:
for pattern #,##0.### the number 123456789.123 can be formatted as 123,456,789.123

If optional negativePattern is absent, then positivePattern is prefixed with the localized minus sign. We use negativePattern to specify negative prefix and suffix. Such characteristics as number of digits, minimal digits and other are the same as in the positivePattern. So we do not obliged to specify negativePattern if it the same as the positivePattern except minus(-) sign.

Digits, separators, prefixes, suffixes and others may be set to arbitrary values. But be careful when setting the strings and symbols. They must be distinct and cannot collide. For example, if group and decimal separators are the same formatter will not be able to parse the number.

Grouping separator is a constant number of digits used to separate thousands or ten-thousands (in some countries). Do not use various number of digits between two group separators.

You can use the following characters in the formatting pattern:

0 a digit - always displayed, even if number has less digits (then 0 is displayed)
# a digit, leading zeroes are omitted
. decimal separator
, grouping separator
E separation of mantissa and exponent for exponential formats
; separates positive and negative subpatterns
- marks the negative number prefix
% multiplies by 100 and shows number as percentage
‰ (\u2030) multiplies by 1000 and shows number as per mille
¤ (\u00A4) currency sign - replaced by the currency sign for the Locale. Also makes formatting use the monetary decimal separator instead of normal decimal separator. ¤¤ makes formatting use international monetary symbols
‘ (\u2018) marks a quote around special characters in prefix or suffix of formatted number
